Project Design Document

Project Overview

Project Name: UNCC Gaming Club Website

Application Description and Purpose

This website is for the UNCC Gaming Club. It helps students find and join gaming events on campus. The site shows information about the club, upcoming game tournaments, and how to become a member.

Intended Users

  • UNCC students who like video games
  • Current club members
  • Students looking for gaming communities
  • Club officers who manage events

Website Content Overview

  • Information about the gaming club
  • List of upcoming gaming events and tournaments
  • Game recommendations and reviews
  • Photo gallery from past events
  • Membership registration form
  • Contact information and social media links

Website Structure

The website has 5 main pages:

  • Home (index.html)
  • About Us
  • Events
  • Game Library
  • Join Us

All pages have a navigation bar at the top and a footer with validation links at the bottom.

Page Design

Page 1: Home (index.html)

  • Purpose: Welcome page that shows what the club is about
  • Audience: All visitors, especially new students
  • Content:
    • Welcome message
    • Latest news and announcements
    • Image slider showing club activities
    • Quick links to events and membership
  • User Input: No data entry on this page
  • Interactive Elements:
    • Image slider (jQuery plugin)
    • Navigation menu
    • Links to other pages
  • Actions: Clicking links takes users to other pages. Image slider changes automatically.

Page 2: About Us

  • Purpose: Tell visitors about the club history and members
  • Audience: New students and potential members
  • Content:
    • Club history and mission
    • Officer profiles with photos
    • Club achievements
    • Meeting schedule
  • User Input: No data entry
  • Interactive Elements:
    • Expandable officer profiles (click to show more info)
    • Photo gallery
  • Actions: Clicking officer cards shows more details using JavaScript

Page 3: Events

  • Purpose: Show upcoming gaming events and tournaments
  • Audience: Current members and interested students
  • Content:
    • Event calendar
    • Event details (date, time, location, game)
    • Past event photos
    • Registration button
  • User Input: Yes, registration form with:
    • Name (required, text validation)
    • Email (required, email format validation)
    • Student ID (required, number validation)
    • Game preference (dropdown)
  • Validations:
    • All fields must be filled
    • Email must have @ symbol
    • Student ID must be numbers only
  • Interactive Elements:
    • Interactive calendar (jQuery-UI datepicker)
    • Photo gallery with lightbox
    • Registration form with validation
  • Actions: Form validates input before submission. Calendar shows event details when clicking dates.

Page 4: Game Library

  • Purpose: Show games the club plays and recommend new games
  • Audience: Members looking for new games to play
  • Content:
    • List of games with images
    • Game ratings and reviews
    • Game categories (Action, RPG, Strategy, etc.)
    • Platform information (PC, Console, Mobile)
  • User Input: No data entry, but users can interact with filters
  • Interactive Elements:
    • Search box to find games
    • Filter dropdowns (by genre, platform)
    • Sort buttons (by rating, by name)
    • Click game card to see details
  • Actions:
    • Search filters game list using JavaScript
    • Dropdown filters update displayed games
    • Sort buttons reorder the list
    • AJAX loads game details without page reload

Page 5: Join Us

  • Purpose: Let students join the club and find contact info
  • Audience: Students who want to become members
  • Content:
    • Membership benefits
    • How to join instructions
    • Application form
    • Social media links
    • FAQ section
  • User Input: Yes, membership form with:
    • Full name (required)
    • Email (required, email validation)
    • Student ID (required, 9 digits)
    • Major (text field)
    • Favorite games (checkboxes)
    • Why join? (text area)
  • Validations:
    • Name cannot be empty
    • Email must be valid format
    • Student ID must be exactly 9 numbers
    • At least one game must be selected
  • Interactive Elements:
    • Form with JavaScript validation
    • FAQ accordion (click to expand answers)
    • Submit button
  • Actions:
    • Form validates all fields before submission
    • Error messages show if validation fails
    • Success message after submission
    • FAQ items expand/collapse when clicked

Dynamic Functionality

1. Image Slider (Home Page)

Description: Automatic slideshow of club photos that users can also control manually.

Technology: jQuery slider plugin or custom JavaScript

Example: Similar to sliders on W3Schools Slideshow

2. Interactive Calendar (Events Page)

Description: Calendar that highlights event dates and shows details when clicked.

Technology: jQuery-UI Datepicker widget

Example: Like calendars on jQuery UI Datepicker Demo

3. Photo Gallery with Lightbox (Events Page)

Description: Click photos to view them larger in a popup overlay.

Technology: Lightbox jQuery plugin

Example: Similar to Lightbox2

4. Form Validation (Events and Join Us Pages)

Description: Check user input before sending. Show error messages for invalid data.

Technology: Custom JavaScript validation

Validations include:

  • Required fields check
  • Email format validation
  • Number-only validation for Student ID
  • Minimum/maximum length checks

5. Search and Filter (Game Library Page)

Description: Real-time search and filtering of game list without page reload.

Technology: JavaScript DOM manipulation

Features:

  • Search box filters as you type
  • Dropdown filters by genre and platform
  • Sort buttons reorder games

6. AJAX Content Loading (Game Library Page)

Description: Load game details without refreshing the whole page.

Technology: jQuery AJAX

Example: Click a game card to load its details from a separate file

7. FAQ Accordion (Join Us Page)

Description: Expandable FAQ items that show/hide answers when clicked.

Technology: jQuery-UI Accordion widget or custom JavaScript

Example: Like jQuery UI Accordion

8. Expandable Officer Cards (About Page)

Description: Click officer photos to see more information.

Technology: JavaScript toggle function

Action: Cards flip or expand to show bio and contact info

Technical Requirements

  • Navigation bar on all pages for single-click access
  • Consistent header and footer across all pages
  • Footer includes HTML and CSS validation links
  • Uses JavaScript/jQuery for interactivity
  • Includes at least one jQuery-UI widget
  • Uses other jQuery plugins (slider, lightbox)
  • Implements AJAX functionality
  • Responsive design for mobile and desktop
